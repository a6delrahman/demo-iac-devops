Git: Die Zeitmaschine

Die Parallelwelt: Versionskontrolle, Branching & Zusammenarbeit im Team.

Pull & Fetch

Team-Abgleich: Änderungen der Kollegen holen.
Pull = Holen & Zusammenführen. Fetch = Nur Holen.

Commit

Speicherpunkt: Einen festen Knoten im Verlauf erstellen. Funktioniert wie "Speichern unter".

Branch

Verzweigung: Sichere, parallele Kopie erstellen. Unabhängiges Arbeiten ohne Störung anderer.

Merge

Zusammenführung: Fertige Arbeit aus einem Branch sicher in einen anderen (z.B. Main) integrieren.

# Lösung 3

IaC & Terraform: Kernkonzepte

Infrastruktur als Code: Wir klicken nicht, wir schreiben Baupläne.

1. Deklarativ (Das "Was")

Wir definieren nur den gewünschten Zielzustand (z.B. "Ich brauche 1 Datenbank"). Terraform entscheidet selbst über das "Wie" (Erstellen, Ändern oder Löschen).

2. Idempotent (Sicherheit)

Das Skript kann beliebig oft ausgeführt werden. Wenn der Zielzustand bereits erreicht ist, macht Terraform einfach nichts. Keine doppelten Server!

3. State (Das Gedächtnis)

Terraform speichert in einem State-File, welche Ressourcen es in der Cloud angelegt hat. So erkennt es Abweichungen ("Drift") sofort.

main.tf (HCL - HashiCorp Configuration Language) 
# Deklaration einer Resource Group
resource "azurerm_resource_group" "rg" {
  name     = "my-app-rg"
  location = "West Europe"
}

# App Service verknüpft mit RG
resource "azurerm_app_service" "web" {
  name                = "frontend-app"
  location            = azurerm_resource_group.rg.location
  resource_group_name = azurerm_resource_group.rg.name
}

# Datenbank
resource "azurerm_sql_database" "db" {
  name                = "app-database"
  resource_group_name = azurerm_resource_group.rg.name
}
